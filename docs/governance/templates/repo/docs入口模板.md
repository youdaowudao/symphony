# docs 入口模板

## 先看哪里

1. `README.md`
2. `SPEC.md`
3. `docs/governance/README.md`
4. `docs/initiatives/README.md`
5. `docs/changes/README.md`
6. `docs/incidents/README.md`

## 文档怎么分流

- 总体系统规格：`SPEC.md`
- 长期主题、阶段规划、长期合同：`docs/initiatives/`
- 单次高风险变更：`docs/changes/`
- 事故与事故级复核：`docs/incidents/`
- 文档治理规则与模板：`docs/governance/`

## 当前启用目录

1. `docs/governance/`
2. `docs/changes/`
3. `docs/incidents/`
4. `docs/initiatives/`

## 备注

- 如果仓库没有系统级长期合同，可以没有 `SPEC.md`
- 如果只有小改动，不必强制创建 change 文档
