Section 3.1 · Project Registry

项目登记表，不是新的 workflow。

这一节要让人类先看懂一件事：多项目不是先加很多 worker，也不是先做新首页。 第一步是让 Symphony 有一张可靠的“项目名单”，并且这张名单成为项目身份的唯一来源。

一句话理解

Project Registry = Symphony 的项目户口本。

它回答：有哪些项目？每个项目对应哪个 Linear project？这个项目是否启用？最多同时跑几个 agent？

原来已经有什么？需要新增什么？

原本已有

单项目配置

现在的系统只有一个 Linear project，所以很多地方默认“不需要问这是哪个项目”。

tracker: kind: linear project_slug: "one-project"
→
需要新增

多项目登记表

未来系统需要知道多个项目，并且每个项目都有稳定身份。

projects: - project_key: "project-a" enabled: true max_concurrent_agents: 15 - project_key: "project-b" enabled: false max_concurrent_agents: 15

Project Registry 只管“项目身份”

它登记什么 人类怎么理解 为什么重要
project_key 项目的 canonical key，V0.1 直接复用 Linear project slug workspace、API、日志、调度 key、Linear 绑定都靠它避免串项目
display_name 从 Linear project name 读出来的展示名 方便观察，不写回配置，也不作为程序判断依据
enabled 这个项目是否参与调度 可以暂停单个项目，不影响其他项目
max_concurrent_agents 这个项目最多同时跑几个 agent 防止一个项目吃完整个共享池，默认建议 15

它不管什么？

正确边界

Workflow 仍然共享

所有项目还是使用同一套 WORKFLOW.md、prompt、hooks、agent/codex 默认规则。

WORKFLOW.md：怎么做事
↓
Project Registry：有哪些项目
错误理解

不是每项目一套 workflow

不要把 Project Registry 做成 per-project workflow 的入口。那会把范围扩大成另一套平台。

Project A：Workflow A
Project B：Workflow B
Project C：Workflow C

为什么不能只在首页加项目标签？

只在 UI 贴标签

看起来知道项目，底层可能不知道

<Issue id="123" label="Project A" />

如果 workspace、retry、cleanup 不知道 project，后面还是可能串线。

从出生就带身份

issue 一被拉取就带 ProjectContext

<Issue> <ProjectKey>project-a</ProjectKey> <DisplayName>项目 A</DisplayName> <IssueId>123</IssueId> </Issue>

这样调度、workspace、日志、API、首页都可以追溯到同一个项目。

这就是 3.1 的核心风险：如果项目身份只在展示层存在，那它不能保护 workspace，也不能保护 cleanup。

配置态和运行态要分开

Project Registry Config

稳定配置

这些是人工配置或启动时规范化出来的项目名单。

project_key: "project-a" enabled: true max_concurrent_agents: 15
Project Runtime State

运行中变化

这些是运行时产生的状态，不应该写回项目登记表。

last_error_class: "rate_limit" backoff_until: "2026-05-24T12:00:00Z" last_poll_ok_at: "..."
人类理解：户口本不要写“今天感冒了”。项目登记表记录身份和配置开关，运行状态记录现在发生了什么。全局默认上限是 20，每项目默认上限是 15。展示名从 Linear project name 读，不写回户口本。

讨论 3.1 时，人类只需要先确认这五件事

01project_key 直接复用哪个 Linear project slug？
02display_name 是否只从 Linear project name 投影？
03旧的 tracker.project_slug 怎么迁移？
04每项目默认并发上限是多少？
05Project Registry 放在哪里？
