# 03. Claude Code History Viewer 试用报告

日期：`2026-05-23`
状态：`已试用`

## 1. 这次试用了什么

这次实际试用的是 `Claude Code History Viewer`。

结论先写在前面：

- **能用，而且很好用。**
- **功能很多，UI 也明显比前面的小工具成熟。**
- **对我现在的需求来说，它已经足够接近“先直接用”的答案。**
- **重新造轮子基本没有必要。**

## 2. 试用后的真实感受

### 2.1 它最强的地方

- 功能很全。
- 会话浏览、搜索、分析、统计、最近编辑这些已经做得很完整。
- 看完以后会明显感觉：很多基础能力别人已经做完了，自己重做不划算。

### 2.2 它的不足

- **默认没有你想要的那种自动实时刷新感。**
- 需要手动刷新。
- UI 偏密，信息堆得比较满，初看不够清晰。
- 详细工具调用需要点进去，不是默认直接摊开。

## 3. 对当前需求的判断

### 3.1 已经满足的部分

对“我要确认 AI 真的在工作”这个目标，它已经够用了。

原因是：

- 能看到会话在持续增长。
- 能看到对话、工具、历史、统计信息。
- 有需要时可以点进去看更细的工具调用。

### 3.2 还差一点的部分

它还不够“专门为 Symphony 定制”。

主要差两点：

- **没有专门只盯某个编排项目的定向展示。**
- **没有默认把当前状态更醒目地显示出来。**

所以它适合先用，但如果你要更贴合 Symphony，后面可能还要做一层轻量适配。

## 4. 关于实时刷新

### 4.1 现在的状态

它支持实时监控会话文件变化，但你试用时感受到的是：

- 不是完全自动地盯着一个你想看的特定目标；
- 更像是“统一扫描 + 手动切换/刷新”的查看器。

### 4.2 可行的补法

你提到的做法是合理的：

- 加一个定时刷新；
- 比如 20 秒或 30 秒刷新一次。

这个属于小改动思路，不是重做。

## 5. 关于目录隔离

这一点很重要。

如果要让第三方工具准确看你指定的记录，**最好把 Codex Home 做隔离**。

也就是说：

- 正常使用的 Codex Home 和专用查看的 Codex Home 分开；
- 专门给查看器看的会话记录单独放；
- 这样第三方工具就不会被别的记录混淆。

这条路是可行的，而且比“让一个工具理解所有历史”更稳。

## 6. 对后续方案的影响

这次试用后的判断已经变了：

- **B 路线是真的可用，不是只停留在理论。**
- **先用现成工具，比立刻自建更合理。**
- **后续如果要做增强，优先做隔离、定向筛选、自动刷新，而不是重造整个 viewer。**

## 7. 仍未解决的问题

你现在真正要的不是“我能不能把每条对话看清楚”，而是：

- 我不需要盯着每条消息；
- 我只需要快速确认后台程序还在跑；
- 我只需要知道它没有卡死、没有摸鱼、没有停住。

按这个标准，`Claude Code History Viewer` 目前**还没有彻底解决问题**，原因很直接：

- 它偏历史浏览；
- 现在主要还是手动刷新；
- 没有默认把“当前活跃状态”做成一个足够醒目的、无损感更强的实时面板。

## 8. 最后结论

`Claude Code History Viewer` 可以作为当前的主观察工具先用起来。  
但如果你的硬要求是“无需盯细节，也能实时确认后台一直在跑”，那它现在还差一截。  
后面要补的不是更多聊天内容，而是更强的运行状态提示。
