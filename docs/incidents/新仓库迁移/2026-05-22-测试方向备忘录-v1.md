# 测试方向备忘录

日期：`2026-05-22`

## 已确定

- 测试噪音必须收口到公共 helper。
- `timeout` 和 `polling` 不再散写在测试正文里。
- 覆盖率门槛必须从 `100` 调整为 `99`。
- 真实业务验证单独保留浏览器层，不混入后台闭环层。
- 浏览器层选型固定为 `Playwright`。
- `MCP 浏览器` 只做人工检查和辅助，不作为正式测试 runner。

## 已检查

- 当前测试流程继续以 `targeted tests` 为默认起点。
- 所有测试命令都必须显式带 `SYMPHONY_TEST_MAX_CASES`。
- `closeout gate`、`fmt/lint`、`make all` 的层级关系保持不变。
- `elixir/TESTING.md` 已覆盖 `polling`、`assert_eventually` 和 `timeout` helper 的基本要求。

## 未来方向

- 真实业务闭环层后续只补当前仓库已经出现、且能直接证明主链的能力。
- `fake` 只做辅助，不再承担主证明层。
