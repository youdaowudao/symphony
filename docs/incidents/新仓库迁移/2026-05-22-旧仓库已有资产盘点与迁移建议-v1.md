# PowerSymphony 旧仓库已有资产盘点与迁移建议 v1

日期：`2026-05-22`
状态：`只读盘点`

## 1. 这份文档做什么

这份文档只做一件事：

- 盘点旧仓库**已经拥有**的资产，并给出迁移建议。

这里的“资产”包括：

- 已有功能
- 已有模块
- 已有接口
- 已有测试资产
- 已有失败经验

这份文档不做：

- 新仓库方案设计
- 新 UI 设计
- 新测试方案设计

后两件事放到另一份文档里。

---

## 2. 先给总判断

旧仓库不是“什么都不行”。

更准确地说：

- **方向层有对的东西**
- **结构层有能用的东西**
- **接口层有可借鉴的东西**
- **实现层尤其是 UI 和测试流里有很多不该继承的东西**

所以对旧仓库最稳的处理不是：

- 整体照搬
- 也不是整体否定

而是：

- 先按资产拆开
- 再逐项判断“保留什么、丢掉什么、只保留理念什么”
