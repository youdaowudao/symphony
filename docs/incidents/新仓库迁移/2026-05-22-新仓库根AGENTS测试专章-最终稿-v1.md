# PowerSymphony 根 AGENTS 测试专章

## 测试

- 测试规则只认本节。
- `WORKFLOW.md` 不写测试制度。
- `elixir/AGENTS.md` 只保留极短执行提醒，或直接回指本节。
- `.github/workflows/*.yml` 和 `Makefile` 只执行命令，不解释制度。

### 本地默认

- 开发时先跑 `targeted tests`。
- 本地默认跑最小 `build` / `fmt-check` / `lint`。
- 本地按需跑真实业务闭环层测试。
- 本地默认不把 `make all` 当开发主路径。
- 本地默认不跑全量 coverage。
- 本地默认不跑 dialyzer。
- 本地默认不跑最重 live E2E。

### 真实业务闭环层

- 仓库必须保留一层本地可跑的真实业务闭环测试。
- 这层优先验证真实业务路径，不优先堆 fake。
- 这层第一版直接服务多项目主线，不围绕单项目扩张。
- 这层默认不依赖真实 Linear。
- 这层默认不依赖真实 Codex 远端。
- 这层默认不依赖真实浏览器。

第一版至少覆盖：

- 多项目静态配置解析与归一化。
- `WORKFLOW.generated.md` 的项目级生成链。
- 多项目 runtime / control-plane 状态收敛。
- 控制面关键 API：`/api/v1/state`、`/api/v1/refresh`、`/api/v1/health`、项目控制、run timeline。
- issue refresh / diff / context summary 的最小结构化闭环。

### 浏览器层

- UI 点击、页面交互、视觉验收不放进真实业务闭环层。
- 浏览器测试如需引入，应独立成层。
- 浏览器层不作为第一版本地默认前置。

### fake

- fake 只用于快反馈、边界条件、难复现异常、小范围状态机验证。
- fake 不作为主正确性证明层。

### 时间断言

- 不要用单点时间断言证明正确性。
- 优先用区间断言。
- 优先用 polling / `assert_eventually`。
- 不要用一次性 `Process.sleep(...)` 后立刻断言异步状态稳定。
- timeout 统一收进公共 helper，不要把毫秒常量散写在测试正文。
- 除非测试目标本身就是极短超时语义，否则不要使用过紧毫秒窗口。

### 本地稳定性

- 本地测试并发必须显式受控。
- 统一通过 `SYMPHONY_TEST_MAX_CASES` 控制并发。
- 必须保留测试噪音治理、cleanup 和重测试资源收敛。

### 远端

- 远端继续保留单条 full gate。
- 远端继续承担 coverage、dialyzer 和完整回归。
- 远端要求不反向绑定本地开发主路径。
