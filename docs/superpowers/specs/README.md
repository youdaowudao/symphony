# Superpowers 设计入口

这里放 Superpowers 方法产出的设计结论与需求快照。

新建 SPEC 时，必须先复制：

- `docs/superpowers/specs/SPEC可验证合同模板.md`

这个目录里的 SPEC，默认承担三件事：

1. 记录稳定的 `目标 / 需求快照`
2. 作为后续 `plans/` 与 `verification/` 的需求对照基线
3. 让 reviewer 不打开 Linear 也能判断实现是否命中需求

每份 SPEC 至少应包含：

1. 用自然语言详细、准确、正面描述功能的 `目标 / 需求快照`
2. 明确的边界与非目标
3. 当前固定约束
4. 可判定的成功标准
5. 风险边界、失败语义与证据映射

写作要求：

- `目标 / 需求快照` 必须先写，且必须用自然语言详细、准确、正面地描述功能。
- 不允许先写实施方案，再反推需求。
- 不允许用术语堆砌代替需求。
- 不允许跳过模板里的任一硬字段。
- 任何缺失字段都意味着这份 SPEC 还不能进入后续 plan。
