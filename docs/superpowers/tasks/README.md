# Superpowers 任务入口

这里放 Superpowers 方法产出的任务拆解与执行记录。

`task` 只做轻量标准化，不做重模板。

任务记录至少要说明：

1. 任务边界
2. 任务依赖
3. 任务状态
4. 当前阻塞
5. 本任务是否允许改生产代码

写作要求：

- 每个 task 只描述一个明确动作。
- 如果 task 是“只改测试”或“只改文档”，必须明确写出禁止碰哪些实现文件。
- 如果 task 发现需要越界，必须暂停并回到 plan / spec。
- 不要把任务写成大而空的 checklist。

推荐结构：

- 任务名
- 目标
- 允许 / 禁止范围
- 当前状态
- 阻塞点
- 下一步
