## 1. 基线验证

已执行：

```bash

SYMPHONY_TEST_MAX_CASES=4 mise exec -- mix test test/symphony_elixir/extensions_test.exs
```

当前状态：

- `/api/v1/state` 已带 `projects[]`
- `running[] / retrying[] / blocked[]` 已带 `project_key`
- state/detail 已统一回退 `project_display_name -> project_key`
- `retrying[]` 现已同步投影 `last_event_at`
- canonical route `/api/v1/projects/:project_key/issues/:issue_identifier` 已兑现
- legacy route 碰到撞名、缺 scope 或 scope 混合冲突都返回 `409 project_scope_required`
- terminal dashboard header 已切到 `enabled / active projects` 摘要，不再保留单项目 tracker link 依赖
- 首页失败态现已兑现“保留上次成功画面 + stale / snapshot unavailable 横幅”
- 首页当前异常区现已兑现 `blocked > retrying > stale` 去重与排序；正常 `running` 区默认按最后更新从旧到新展示

这意味着 `V0.17` 不再卡在“字段不足，回到上游 contract 主题”。

## 2. allowlist

首页当前允许稳定消费：

- 顶层：`generated_at`、`counts.*`、`codex_totals.*`、`rate_limits`、`projects[]`
- `running[]`：`project_key`、`project_display_name`、`issue_id`、`issue_identifier`、`state`、`runtime_status`、`worker_host`、`workspace_path`、`session_id`、`turn_count`、`last_event`、`last_message`、`started_at`、`last_event_at`、`tokens.*`
- `retrying[]`：`project_key`、`project_display_name`、`issue_id`、`issue_identifier`、`attempt`、`due_at`、`last_event_at`、`error`、`worker_host`、`workspace_path`
- `blocked[]`：`project_key`、`project_display_name`、`issue_id`、`issue_identifier`、`state`、`runtime_status`、`error`、`worker_host`、`workspace_path`、`session_id`、`blocked_at`、`last_event`、`last_message`、`last_event_at`
- 页面本地纯显示推导：基于 `started_at` 的 runtime、基于 `last_event_at / blocked_at` 的相对时间、基于 `RuntimeStatus.classify/2` 的 stale 重判定

## 3. stop list

当前首页仍禁止消费或猜测：

- `title`
- `thread_id`
- `turn_id`
- 完整事件流
- `blocked_by` 真正 payload
- `todo pool` 后台数据
- 任何靠 `workspace_path`、`session_id`、路径模式推出来的伪项目身份
- 任何第二状态源、额外 fetch、per-row detail fetch

## 4. detail route verdict

当前 canonical detail route **已兑现**。

证据：

- `elixir/lib/symphony_elixir_web/router.ex`
- `elixir/lib/symphony_elixir_web/controllers/observability_api_controller.ex`
- `elixir/lib/symphony_elixir_web/presenter.ex`
- `elixir/test/symphony_elixir/extensions_test.exs`

结论：

- 首页 row link 可以直接使用 canonical project-scoped route
- 本轮不再属于 `route gap`

## 5. go / no-go 结论

本轮 Task 0 结论固定为：

- `既有字段足够进入 V0.17 UI 实现`

原因：

- `V0.15` 的项目身份 outward contract 已兑现
- `projects[]` 已兑现且保留 enabled 但 0 计数项目
- canonical detail route 已兑现
- `workspace.path` 猜路径 fallback 已移除
- `project_display_name` 已收窄为只读展示回退，不再依赖第二真相源
- 首页异常区与正常 `running` 区所需的排序与相对时间输入已经足够，不需要额外 fetch 或第二状态源

## 6. 进入实现后的边界提醒

- `V0.17` 仍然不能重开 `V0.15/V0.16` contract
- `todo pool / blocked-by` 仍只能是手动检查占位区
- 异常/正常 running 分栏必须用当前 `@now` 做 stale 重判定，不能只信 mount 时固化的 payload
