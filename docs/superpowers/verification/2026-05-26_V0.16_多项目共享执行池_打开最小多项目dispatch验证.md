  test/symphony_elixir/tracker_project_aggregation_test.exs \

  test/symphony_elixir/status_dashboard_snapshot_test.exs

通过标准：

- 单项目 fetch failure 不阻断其他项目。
- registry 解析失败、legacy slug / registry 冲突、`WORKFLOW.md` 无效这类共享配置失败会阻断全池新 dispatch。
- `cleanup_failed` 只影响当前 issue。
- 单 host failure 只影响该 host 新 dispatch。
- rate limit 只阻断受影响范围的新 Codex start。
- `WORKFLOW.md` 无效或所有项目 fetch 全失败时，全池新 dispatch 被阻断。
- 若当前 outward surface 只能证明“失败事实仍可定位”，而不能直接表达某类新失败范围，阶段性验收记录必须把该项标为未闭环，不得写成“已完成”。
