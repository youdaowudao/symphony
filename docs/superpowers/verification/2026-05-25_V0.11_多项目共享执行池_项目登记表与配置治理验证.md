## 0. 本文件为什么存在

本文件不是第二份 SPEC，也不是 implementation plan。它只记录一次文档阶段复核结论：V0.11 是否已经从过宽的“registry/config + ProjectContext + Linear fetch”收回到“只做 registry/config”。

保留它的价值：

- 给后续 coder 和 reviewer 一个快速入口，知道本次为什么移出 ProjectContext / tracker 聚合。
- 防止后续实现时把已经排除的 `Linear.Client.fetch_candidate_issues/0`、`ProjectContext` 模块、dashboard / API 改造又塞回 V0.11。
- 记录未解决风险，明确这些风险应进入后续主题，而不是被误认为已经在 V0.11 解决。

它不承载长期配置合同。长期合同仍以 V0.1 主规划、`03_配置治理合同.md`、`04_配置模板.md`、`05_实现主题拆分与后续候选.md` 和本主题 SPEC / PLAN 为准。

如果后续发现本文件和 SPEC / PLAN 冲突，必须改 SPEC / PLAN 或更新本验证记录，不能让验证记录成为第三份真相源。
