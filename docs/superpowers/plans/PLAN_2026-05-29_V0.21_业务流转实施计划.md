# V0.21 业务流转实施计划 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
> Status note: 本计划已完成文档阶段 Level 3 复核，可作为进入实现阶段的正式依据；若后续实现触发本文停止条件，必须先回到对应 SPEC 再继续。

**Goal:** 把 V0.21 业务流转合同落成一组可执行改动，覆盖项目级 review 配置、`review_attempt_limit` 计数边界、`Done-only` / merged-fact 收口判定、`AI Review` prompt / workflow、文档同步和实现前验证。

**Architecture:** 先补齐运行期配置链路，让 `review_mode` 与 `review_attempt_limit` 能进入 `ProjectContext`；紧接着把 `review_attempt_limit` 的计数边界、`Rework` reset 和封顶出口写成可执行合同；再修正 orchestrator 的 `Done-only` blocker 语义与 `Done` 只认当前 cycle 当前有效 PR/head merged 事实的收口语义；之后重写 `WORKFLOW.md` 的 active states、reviewer 前缀、Submission Refresh Gate 与 `Human Review` / `Rework` 行为；最后同步治理文档、长期 SPEC 和验证文档，确保仓库不再同时保留两套主线。整个计划默认走 `closeout gate`，只跑 targeted tests 与 `fmt/lint`；如果执行过程中发现必须引入第二 reviewer workspace、自动 merge 接手角色或新的 orchestrator 评论聚合链路，立即停下回到 SPEC。

**Tech Stack:** Elixir / OTP、`SymphonyElixir.ProjectRegistry`、`SymphonyElixir.ProjectContext`、`SymphonyElixir.Tracker.ProjectAggregation`、`SymphonyElixir.Orchestrator`、`SymphonyElixir.PromptBuilder`、`elixir/WORKFLOW.md`、`AGENTS.md`、`elixir/CHANGE_FLOW.md`、`docs/governance/*`、ExUnit、`SYMPHONY_TEST_MAX_CASES=4`。
