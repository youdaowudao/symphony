# 多项目共享执行池 V0.15 最小观测 gate Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
> Status note: 本计划当前是 draft execution candidate；只有在对应 SPEC 通过当前 `Level 3` 文档复核后，才可作为稳定执行依据。

**Goal:** 把 `V0.14` 的 project-aware runtime state 投影到共享 observability contract 上，补齐 snapshot、runtime outward messages、state payload、project-scoped detail route、Web dashboard 和 terminal dashboard 的最小多项目观测能力，同时保持 dispatch / workspace / rollout 边界不变。

**Architecture:** 先只读核对当前 repo 的 `snapshot -> runtime outward messages -> Presenter -> Router/API -> LiveView -> terminal dashboard` 现状，再按共享 contract 的顺序推进：先锁定 snapshot / runtime message / Presenter / route 语义，再改 Web dashboard 和 terminal dashboard，最后做 allowlist diff、关键 hunk 复核与回归验证。整个计划禁止把阶段 5 扩大成 dispatch rollout、workspace contract 修复或 UI 模板确认。

**Tech Stack:** Elixir / OTP、Phoenix Router、Phoenix Controller、Phoenix LiveView、`SymphonyElixir.Orchestrator`、`SymphonyElixir.AgentRunner`、`SymphonyElixir.ProjectRegistry`、`SymphonyElixirWeb.Presenter`、`SymphonyElixir.StatusDashboard`、嵌入式静态资源、ExUnit、`SYMPHONY_TEST_MAX_CASES=4`。
